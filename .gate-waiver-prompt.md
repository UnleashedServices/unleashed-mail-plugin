## The situation

§2 promises a user-authorized waiver so the gate "must NOT hard-wedge the dev loop with no escape".
**No component implements it** — `review-synthesis` never emits the `WAIVED:` marker and
`review-verdict.py` has no `WAIVED` concept, while `_quorum_problem` demands gemini **and** codex each
APPROVING. Net: a waived reviewer can never produce an approving artifact → `implement` blocks forever →
the exact hard-wedge §2 forbids. **Verify this claim yourself before reviewing the options** — if the
waiver is in fact implemented somewhere I missed, say so and stop; that changes everything.

## The crux

§2 says **only the user may waive — never an agent, never automatic.** The plan argues no script can
enforce that (the agent is the process invoking the script; any flag a human could type, the agent can
type unprompted), so a `WAIVED:` line is **attestation, not enforcement** — and an unenforceable claim
dressed as a mechanical control is a **false audit trail**.
