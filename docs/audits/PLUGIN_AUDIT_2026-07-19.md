### MAJ-9: `skills/implement/SKILL.md:51` — security

The Design Gate bash script splices $ARGUMENTS textually into double-quoted shell context (lines 51, 66), so an argument containing ", $( ), or backticks is executed as shell — a fifth bypass of the containment guard the script hardens four other ways — and innocent arguments with quotes break plan resolution.

- **Why it matters:** Per live skills docs, '$ARGUMENTS always expands to the full argument string as typed' before the model sees the body. `/implement x"; touch /tmp/pwned; _="` renders `if [ -f "x"; touch /tmp/pwned; _="" ]` and executes the injected command; `/implement plan" ] || PLAN=docs/planning/ANY_PLAN.md #` can force PLAN past the exact-match resolution the comments spent four review rounds hardening (symlink, .., realpath, tr-range fixes all assume the argument reaches the script inertly). Even benignly, a feature name containing a double quote or `$` makes the gate script error out.
- **Suggested fix:** Never interpolate $ARGUMENTS into shell syntax. Bind it once via a quoted heredoc at the top of the block: `ARG=$(cat <<'UM_EOF'\n$ARGUMENTS\nUM_EOF\n)` then use "$ARG" everywhere (and reject multi-line values), or have the skill write the argument to a temp file that the script reads.
- **Verification:** implement/SKILL.md lines 51-52 and 66 splice $ARGUMENTS directly into double-quoted shell, and live docs confirm '$ARGUMENTS always expands to the full argument string as typed' before the body is rendered — so a double quote, $( ), or backtick in the argument is executed as shell or breaks the gate script, defeating the containment guard the same script hardens against four other bypasses. Both the injection and the benign-breakage failure modes are real and reachable.

- **Verification:** Verified all cited lines: both review skills, review-synthesis, and brainstorm hard-code /tmp/agy-out.txt and /tmp/codex-out.txt with no pre-clean instruction anywhere (grep across all six cited files); review-verdict.py's provenance checks are purely intra-artifact with no mtime/freshness check, the snapshot binds only plan bytes, and pty-capture.py writes the transcript only at run end, so a never-started or killed wrapper leaves the previous round's APPROVE transcript to satisfy the gate.
